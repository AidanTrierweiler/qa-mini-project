QA Manual & API Testing Mini Project:

This repository demonstrates basic Quality Assurance testing skills including manual UI testing, bug reporting, and API testing using Postman.


Project Overview:

The goal of this project is to simulate real QA work by testing a sample web application and a public API.
All testing activities were documented using structured test cases and reproducible bug reports.


Tools Used:

Manual Browser Testing
Postman (API Testing)
Markdown Documentation
GitHub (Version Control & Documentation)


1. Manual UI Testing
Application Tested:
https://the-internet.herokuapp.com/login

What was tested:

Valid login
Invalid credentials
Empty input fields
Logout functionality

Documentation:

test-cases.md — step-by-step functional test cases
bug-reports.md — documented defects with expected vs actual behavior


2. API Testing
API Tested:
https://jsonplaceholder.typicode.com

Endpoints Covered:
GET /users
GET /posts
GET /posts?userId=1 (filtered results)
POST /posts (create resource)
DELETE /posts/1 (remove resource)

Validations Performed:
HTTP status codes
Response structure
Field presence
Basic data behavior
Documentation available in api-test-cases.md


Skills Demonstrated:

Writing structured test cases
Identifying and documenting bugs
Understanding client vs server behavior
API request testing (GET, POST, DELETE)
Reading JSON responses
Organizing QA documentation professionally
